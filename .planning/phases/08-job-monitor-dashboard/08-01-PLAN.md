## User Decisions (from CONTEXT.md)

**Job List Layout:**
- Table view with columns: Job ID, Type, Status, Created Time, Actions
- Compact row height for scanning many jobs
- Sortable by time (newest first by default)

**Auto-Refresh Behavior:**
- Pause polling when tab hidden using Page Visibility API
- Poll every 2 seconds when Jobs tab is active AND visible
- Display "Last updated: X seconds ago" indicator
- Manual refresh button available

**Job Detail Display:**
- Expand row inline (accordion style)
- Click job row to expand/collapse details
- Expanded view shows: full request params, result (if completed), error (if failed), timestamps
- Image preview inline for completed generation jobs
- Only one job expanded at a time

**Status Visualization:**
- Colors + icons for all 5 states:
  * Pending: Yellow/amber + hourglass icon
  * Processing: Blue + animated spinner icon
  * Completed: Green + checkmark icon
  * Failed: Red + X icon
  * Cancelled: Gray + ban icon

**Cancel Button Behavior:**
- Show only for Pending and Processing jobs
- Confirmation dialog before canceling
- Update UI immediately on cancel (optimistic)

## API Job Model

```javascript
{
  id: string (GUID),
  type: "text2img" | "img2img" | "upscale",
  status: "Pending" | "Processing" | "Completed" | "Failed" | "Cancelled",
  createdAt: ISO timestamp,
  startedAt: ISO timestamp | null,
  completedAt: ISO timestamp | null,
  progress: 0-100,
  result: { image: "base64..." } | null,
  error: string | null,
  request: object
}
```

## API Endpoints

- GET /api/jobs — List all jobs (array of Job objects)
- GET /api/jobs/{id} — Get specific job details
- DELETE /api/jobs/{id} — Cancel job (returns success/failure)
