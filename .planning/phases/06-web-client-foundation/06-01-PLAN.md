## Key Context from User Decisions

**Server Architecture:**
- Separate ASP.NET Core static file server (not embedded in API)
- Port 5001 (distinct from API on 5000)
- Uses ASP.NET Core's built-in static file middleware
- Serves files from wwwroot folder

**UI Framework:**
- Pure vanilla HTML/CSS/JS — NO build step, NO frameworks
- No external CSS frameworks (hand-written CSS)
- Modern ES6+ JavaScript
- Fetch API for HTTP requests

**Layout:**
- Sidebar navigation with sections: Generate, Jobs, Models, Inspector
- Fixed left sidebar, main content area
- Responsive: sidebar collapses on mobile

**API Connection:**
- Hardcoded localhost:5000 (simplest for local dev)
- Display connection status indicator

## Existing Code Patterns (from Amuse.API)

- **Minimal APIs** pattern with top-level statements
- **Configuration via appsettings.json** for ports/settings
- **Serilog** for structured logging
- **Project structure**: Separate .csproj, Program.cs entry point

## API Endpoints to Connect To

The web client will eventually call these API endpoints:
- GET /health — Connection check
- GET /api/info — API info
- POST /api/generate/text2img — T2I generation
- POST /api/generate/img2img — I2I generation  
- POST /api/upscale — Upscale
- GET /api/jobs — List jobs
- GET /api/models — List models
- GET /api/models/loaded — Loaded models
