## User Decisions (from CONTEXT.md)

**Model List Layout:**
- Card view (Option A) — Grid layout, responsive (2 columns desktop, 1 mobile)
- Cards show: Model name, type, size, format, load status
- Visual distinction between loaded and unloaded models

**Loaded Model Indication:**
- Status badge + visual highlight (Option B)
- Green "Loaded" badge for loaded models
- Subtle background tint for loaded model cards
- Memory usage and load timestamp shown only for loaded models

**Load/Unload Controls:**
- Buttons in each model card (Option B)
- "Load" button (primary style) for unloaded models
- "Unload" button (warning style) for loaded models
- Button disabled and shows spinner during operation
- Confirmation dialog before unload

**Memory Display:**
- Human-readable format (MB/GB)
- Progress bar or badge showing memory usage
- Total loaded memory summary at top of page

**Polling Strategy:**
- Poll when Models tab is active (same pattern as Jobs)
- 5-second interval (models change less frequently than jobs)
- Pause when tab hidden (Page Visibility API)
- Manual refresh button available

## API Model Data Structure

```javascript
{
  id: string,           // e.g., "sd-1-5"
  name: string,         // e.g., "Stable Diffusion 1.5"
  type: string,         // "StableDiffusion" | "Upscaler"
  path: string,         // File path
  format: string,       // "ONNX" | "Safetensors"
  size: number,         // Size in bytes
  description: string,  // Optional
  isLoaded: boolean,    // Currently loaded?
  loadedAt: string,     // ISO timestamp when loaded
  memoryUsage: number   // Memory in bytes when loaded
}
```

## API Endpoints

- GET /api/models — List all models
- POST /api/models/{id}/load — Load model into memory
- POST /api/models/{id}/unload — Unload model from memory
