<objective>
Create the Amuse.API console project with basic server running. This is Phase 1 of building the headless API for AmuseAI.

Purpose: Establish the foundation for all subsequent API phases. The server must be able to start, respond to health checks, and provide API information.

Output: Working API server that listens on configurable port, responds to /health and /api/info endpoints.
</objective>

@.planning/phases/01-api-server-foundation/01-RESEARCH.md

**Key decisions from CONTEXT:**
- Standalone solution (Amuse.API.sln) separate from main Amuse.sln
- Minimal JSON response format
- JSON error bodies with details
- Serilog (same as Amuse.UI) — console sink + file sink

**From RESEARCH:**
- Use WebApplication.CreateBuilder(args) pattern
- Two-stage Serilog initialization (bootstrap + full)
- Port via appsettings.json "Urls" key or --urls command line
- Graceful shutdown handled natively by WebApplication.Run()
</context>
